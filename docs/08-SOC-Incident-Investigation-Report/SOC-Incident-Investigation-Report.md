# BlueSentinel SOC Lab
# Incident Investigation Report

---

## Incident Summary

This report documents the investigation performed within the BlueSentinel SOC Lab following the simulation of reconnaissance activity against a monitored Windows endpoint. The investigation combined endpoint telemetry, network packet analysis, and centralized security monitoring to evaluate system activity and document the incident using a structured Security Operations Center (SOC) methodology.

The objective of this investigation was to analyze the generated security telemetry, validate monitoring capabilities, examine collected evidence, and document the findings obtained throughout the investigation lifecycle.

# Investigation Methodology

The investigation followed a structured SOC workflow consisting of multiple analytical stages.

1. Establish a secure monitoring environment.
2. Verify endpoint telemetry collection.
3. Capture baseline network activity.
4. Identify available network assets.
5. Simulate reconnaissance activity.
6. Analyze captured network traffic.
7. Review security events within Wazuh.
8. Correlate collected evidence.
9. Document findings and recommendations.

---

# Attack Overview

Reconnaissance activity was simulated from the Kali Linux system using multiple Nmap scanning techniques against the monitored Windows endpoint.

The generated traffic was intentionally designed to emulate the initial reconnaissance phase of the cyber attack lifecycle, allowing both network and endpoint monitoring components to collect security telemetry for investigation.

# Wireshark Analysis

Network traffic generated during reconnaissance activity was analyzed using Wireshark to examine packet-level communication between the attacker and monitored endpoint.

The analysis included:

- TCP three-way handshake
- SYN packet identification
- Source and destination IP analysis
- Port communication
- Protocol behavior
- Connection attempts

Packet analysis provided visibility into the reconnaissance activity occurring across the monitored network.

---

# Wazuh Investigation

Security events generated by the monitored endpoint were investigated through the Wazuh Threat Hunting interface.

The investigation included analysis of:

- Security event metadata
- Alert information
- Endpoint details
- Rule metadata
- Raw JSON events
- Event correlation
- Investigation timeline

The collected telemetry provided centralized visibility into monitored endpoint activity and demonstrated the alert investigation workflow within the SIEM platform.

---

# Findings

The investigation resulted in the following observations:

- Endpoint telemetry was successfully collected through Sysmon.
- Network traffic generated during reconnaissance activity was successfully captured using Wireshark.
- Asset discovery accurately identified active hosts and available services within the lab environment.
- Wazuh provided centralized visibility into endpoint security events and supported structured alert investigation.
- Correlation of endpoint telemetry and network evidence improved overall investigation visibility.

---

# Recommendations

Based on the investigation, the following recommendations are proposed:

- Continuously monitor endpoint telemetry using centralized SIEM solutions.
- Regularly establish network baselines to improve anomaly detection.
- Perform periodic asset discovery to maintain network visibility.
- Review security alerts using structured triage procedures.
- Maintain comprehensive technical documentation for future investigations.
- Continuously evaluate and improve monitoring configurations.
